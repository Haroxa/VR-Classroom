## 通用说明

### 基础URL

```
开发环境: http://localhost:8080
生产环境: https://your-domain.com
```

### 认证方式

使用JWT Token认证，在请求头中添加：

```
Authorization: Bearer <token>
```

### 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

### 错误码说明

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证/认证失败 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 资源冲突 |
| 500 | 服务器内部错误 |

## 用户模块

### 1. 创建用户

**接口**: `POST /api/users`

**描述**: 创建新用户（主要用于测试环境）

**认证**: 无需认证

**请求参数**:

```json
{
  "phone": "13800138000",
  "name": "用户名",
  "avatar": "头像URL",
  "openId": "微信openId（可选）"
}
```

**执行逻辑**:

```
1. 接收用户信息
2. 如果没有openId，自动生成默认openId
3. 调用UserService.saveUser()保存用户
4. 返回用户ID
```

**响应示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1
  }
}
```

### 2. 微信登录

**接口**: `POST /api/users/login`

**描述**: 微信小程序登录（v4.1）

**认证**: 无需认证

**请求参数**:

```json
{
  "loginCode": "微信登录code",
  "phoneCode": "手机号code",
  "nickName": "用户昵称（可选）",
  "avatarUrl": "用户头像URL（可选）"
}
```

**执行逻辑**:

```
1. 接收微信登录code和手机号code
2. 调用微信API获取openId和sessionKey
   └─ 调用WechatUtil.getOpenIdAndSessionKey(loginCode)
3. 调用微信API获取用户手机号
   └─ 调用WechatUtil.getPhoneNumber(phoneCode, sessionKey)
4. 根据手机号查询用户
   ├─ 如果用户存在，更新用户信息（昵称、头像）
   └─ 如果用户不存在，创建新用户
       ├─ 设置openId
       ├─ 设置手机号
       ├─ 设置昵称（如果提供）
       ├─ 设置头像（如果提供）
       └─ 设置认证状态为0
5. 生成JWT Token
6. 返回Token和用户信息
```

**流程图**:

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  前端请求   │ ──▶ │  调用微信API  │ ──▶ │  获取openId │
└─────────────┘     └──────────────┘     └─────────────┘
                                               │
                                               ▼
                              ┌──────────────┐
                              │ 获取手机号   │
                              └──────────────┘
                                               │
                                               ▼
                    ┌──────────────────────────────────────┐
                    │           根据手机号查询用户        │
                    └──────────────────────────────────────┘
                              │                │
                         用户存在          用户不存在
                              │                │
                              ▼                ▼
                    ┌──────────────┐   ┌──────────────┐
                    │  更新用户    │   │  创建新用户  │
                    │  信息        │   │              │
                    └──────────────┘   └──────────────┘
                              │                │
                              └───────┬────────┘
                                      ▼
                              ┌──────────────┐
                              │  生成Token   │
                              │  返回用户信息 │
                              └──────────────┘
```

**响应示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": 1,
      "name": "微信用户",
      "phone": "13800138000",
      "avatar": "https://example.com/avatar.png",
      "verifyStatus": 0
    }
  }
}
```

### 2. 创建帖子

**接口**: `POST /api/posts`

**描述**: 创建新帖子

**认证**: 需要JWT Token

**请求参数**:

```json
{
  "title": "帖子标题",
  "content": "帖子内容",
  "categoryId": 1,
  "images": ["图片URL1", "图片URL2"]
}
```

**执行逻辑**:

```
1. 从请求属性中获取用户ID
2. 如果用户ID为空，返回401错误
3. 参数验证
   ├─ 标题不能为空
   ├─ 内容不能为空
   └─ 分类ID不能为空
4. 调用PostService.createPost()
   ├─ 创建帖子记录
   │  ├─ 设置标题、内容、分类
   │  ├─ 设置作者ID
   │  ├─ 设置状态为待审核(0)
   │  └─ 设置创建时间
   ├─ 如果有图片，保存图片记录
   └─ 返回帖子ID
5. 返回创建结果
```

**流程图**:

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  前端请求   │ ──▶ │  验证用户    │ ──▶ │  参数验证   │
└─────────────┘     └──────────────┘     └─────────────┘
                                               │
                                               ▼
                    ┌──────────────────────────────────────┐
                    │           创建帖子记录               │
                    │  ┌─────────────────────────────────┐ │
                    │  │ title, content, categoryId     │ │
                    │  │ authorId, status=0, createdAt  │ │
                    │  └─────────────────────────────────┘ │
                    └──────────────────────────────────────┘
                                      │
                                      ▼
                              ┌──────────────┐
                              │  保存图片    │
                              │  (如果有)    │
                              └──────────────┘
                                      │
                                      ▼
                              ┌──────────────┐
                              │  返回帖子ID  │
                              └──────────────┘
```

**响应示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": "1"

```

```
   │  └─ 更新修改时间
```


## 订单模块

### 1. 获取教室座位图

**接口**: `GET /api/rooms/{roomId}/seats`

**描述**: 获取指定教室的座位布局和状态

**认证**: 无需认证

**路径参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| roomId | String | 是 | 教室ID |

**执行逻辑**:

```
1. 接收教室ID
2. 尝试从Redis缓存获取
   ├─ 如果缓存命中，直接返回
   └─ 如果缓存未命中，继续执行
3. 调用OrderService.getRoomSeats()
   ├─ 查询教室信息
   │  ├─ 教室名称
   │  ├─ 总行数
   │  └─ 总列数
   ├─ 查询教室所有座位
   │  ├─ 座位ID
   │  ├─ 座位位置(row, col)
   │  ├─ 座位状态
   │  │  ├─ 1: 可选
   │  │  ├─ 2: 锁定
   │  │  └─ 3: 已购买
   │  └─ 捐赠者信息（如果已购买）
   └─ 组装座位图数据
4. 写入Redis缓存（过期时间5分钟）
5. 返回座位图数据
```

**座位状态说明**:

| 状态值 | 说明 | 前端显示 |
|--------|------|----------|
| 1 | 可选 | 绿色/白色 |
| 2 | 锁定 | 黄色/灰色 |
| 3 | 已购买 | 红色/深色 |

**响应示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "roomId": 1,
    "roomName": "教室A101",
    "totalRows": 10,
    "totalCols": 20,
    "seats": [
      {
        "id": 1,
        "row": 1,
        "col": 1,
```

用于防止同一资源被并发操作：

```java
String lockKey = "seat:lock:" + seatId;
if (!distributedLock.tryLock(lockKey, 30, TimeUnit.SECONDS)) {
    throw new BusinessException(409, "资源正在被处理");
}
try {
    // 业务逻辑
} finally {
    distributedLock.unlock(lockKey);
}
```
