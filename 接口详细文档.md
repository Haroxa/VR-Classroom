# VR教室系统接口详细文档

## 1. 项目概述

VR教室系统是一个基于Spring Boot的后端服务，提供用户管理、座位预约、论坛交流等功能。系统采用模块化设计，主要包括用户模块、订单模块、论坛模块、后台管理模块等。

## 2. 模块划分

| 模块名称 | 主要功能 | 状态 |
|---------|---------|------|
| user | 用户管理、登录认证 | 活跃 |
| order | 座位预约、订单管理 | 活跃 |
| forum | 帖子、评论管理 | 活跃 |
| admin | 后台管理、内容审核 | 活跃 |
| space | 空间模型（仅model和mapper） | 辅助模块 |
| common | 公共功能、文件上传 | 活跃 |
| test | 测试接口 | 开发环境 |

## 3. 核心接口文档

### 3.1 用户模块 (User)

#### 3.1.1 创建用户
- **接口地址**: `POST /api/users`
- **请求参数**:
  - `openId`: String (可选，默认生成)
  - `phone`: String (手机号)
  - `name`: String (用户名)
  - `avatar`: String (头像地址)
  - `collegeId`: String (学院ID)
- **返回参数**:
  - `id`: Integer (用户ID)

#### 3.1.2 微信登录
- **接口地址**: `POST /api/users/login`
- **请求参数**:
  - `loginCode`: String (微信登录码)
  - `phoneCode`: String (手机号授权码)
  - `nickName`: String (微信昵称)
  - `avatarUrl`: String (微信头像)
- **返回参数**:
  - `token`: String (JWT令牌)
  - `user`: Object (用户信息)

#### 3.1.3 手机号登录
- **接口地址**: `POST /api/users/login/phone`
- **请求参数**:
  - `phone`: String (手机号)
- **返回参数**:
  - `token`: String (JWT令牌)
  - `user`: Object (用户信息)

#### 3.1.4 获取用户信息
- **接口地址**: `GET /api/users/profile`
- **请求头**:
  - `Authorization`: Bearer {token}
- **返回参数**:
  - `id`: String (用户ID)
  - `name`: String (用户名)
  - `avatar`: String (头像地址)
  - `phone`: String (手机号)
  - `collegeId`: String (学院ID)
  - `collegeName`: String (学院名称)
  - `verifyStatus`: Integer (认证状态)

#### 3.1.5 获取用户帖子
- **接口地址**: `GET /api/users/posts`
- **请求头**:
  - `Authorization`: Bearer {token}
- **请求参数**:
  - `page`: Integer (页码，默认1)
- **返回参数**:
  - `current`: Integer (当前页码)
  - `total`: Integer (总页数)
  - `records`: Array (帖子列表)

#### 3.1.6 获取用户评论
- **接口地址**: `GET /api/users/comments`
- **请求头**:
  - `Authorization`: Bearer {token}
- **请求参数**:
  - `page`: Integer (页码，默认1)
- **返回参数**:
  - `current`: Integer (当前页码)
  - `total`: Integer (总页数)
  - `records`: Array (评论列表)

### 3.2 订单模块 (Order)

#### 3.2.1 获取教室座位信息
- **接口地址**: `GET /api/rooms/{roomId}/seats`
- **路径参数**:
  - `roomId`: String (教室ID)
- **返回参数**:
  - `totalRows`: Integer (总排数)
  - `totalCols`: Integer (总列数)
  - `seats`: Array (座位列表)
- **技术实现**:
  - 使用Redis缓存优化查询性能，缓存时间5分钟
  - 缓存键格式：`room:seats:{roomId}`

#### 3.2.2 创建订单
- **接口地址**: `POST /api/orders`
- **请求头**:
  - `Authorization`: Bearer {token}
- **请求参数**:
  - `campusId`: Integer (校区ID)
  - `buildingId`: Integer (楼栋ID)
  - `roomId`: Integer (教室ID)
  - `seatList`: Array (座位列表)
    - `id`: String (座位ID)
    - `version`: Integer (版本号)
- **返回参数**:
  - `orderId`: String (订单ID)
- **技术实现**:
  - 使用Redis分布式锁锁定座位，锁前缀：`seat:lock:`
  - 使用乐观锁更新座位状态，防止并发冲突
  - 订单过期时间：10分钟
  - 价格单位统一使用"分"

#### 3.2.3 更新订单
- **接口地址**: `PATCH /api/orders/{orderId}`
- **请求头**:
  - `Authorization`: Bearer {token}
- **路径参数**:
  - `orderId`: String (订单ID)
- **请求参数**:
  - `status`: String (订单状态，如CANCELLED)
- **返回参数**:
  - 空对象
- **技术实现**:
  - 取消订单时自动释放座位
  - 使用乐观锁更新座位状态

#### 3.2.4 模拟支付回调
- **接口地址**: `POST /api/mock/pay/notify`
- **请求参数**:
  - `orderId`: String (订单ID)
- **返回参数**:
  - 空对象
- **技术实现**:
  - 支付成功后将座位状态更新为已售出（status=3）

#### 3.2.5 获取订单列表
- **接口地址**: `GET /api/orders`
- **请求头**:
  - `Authorization`: Bearer {token}
- **请求参数**:
  - `page`: Integer (页码，默认1)
  - `size`: Integer (每页数量，默认10)
- **返回参数**:
  - `current`: Integer (当前页码)
  - `total`: Integer (总页数)
  - `records`: Array (订单列表)

## 5. 技术实现细节

### 5.1 价格单位统一
- **实现**: 所有价格字段统一使用"分"作为单位
- **配置**: `application.yml` 中 `order.seat.price` 配置为 100000 (1000元)
- **计算**: 直接使用分进行价格计算，无需单位转换
- **状态**: ✅ 已完成

### 5.2 分布式锁
- **实现**: 使用Redis分布式锁处理并发场景下的座位锁定
- **超时时间**: 30秒
- **锁前缀**: `seat:lock:`
- **使用场景**: 订单创建时锁定座位

### 5.3 乐观锁
- **实现**: 使用版本号机制实现座位状态更新的并发控制
- **字段**: `version` 字段用于乐观锁控制
- **使用场景**: 座位状态更新（锁定、释放、确认）

### 5.4 分页查询
- **实现**: 使用MyBatis-Plus的Page和IPage实现数据库层面的分页查询
- **默认页大小**: 20条

### 5.5 Redis缓存
- **实现**: 使用Redis缓存教室座位信息
- **缓存时间**: 5分钟
- **缓存键**: `room:seats:{roomId}`
- **状态**: ✅ 已完成

### 5.6 批量查询优化
- **实现**: 使用MyBatis-Plus的selectBatchIds进行批量查询
- **优化场景**: 后台管理模块的帖子和评论列表查询
- **优化效果**: 将N+1查询优化为批量查询，减少数据库访问次数
- **状态**: ✅ 已完成

### 5.7 统一异常处理
- **实现**: 使用统一的BusinessException处理业务异常
- **错误码**: 定义统一的错误码规范
- **使用场景**: 帖子审核、评论审核等业务操作
- **状态**: ✅ 已完成

### 5.8 参数校验
- **实现**: 使用JSR-303注解进行声明式参数校验
- **注解**: `@NotNull`、`@NotBlank`、`@Size`、`@Pattern`、`@Min`、`@Max`
- **校验范围**: PostCreateDTO、PostUpdateDTO、CommentCreateDTO、CommentUpdateDTO、LoginDTO
- **异常处理**: GlobalExceptionHandler统一处理MethodArgumentNotValidException
- **状态**: ✅ 已完成

### 5.9 审计日志
- **实现**: 使用AOP切面实现统一的审计日志记录
- **注解**: `@AuditLog`，支持配置操作名称、模块、是否记录参数/返回值
- **切面**: `AuditLogAspect`，自动记录操作人、操作时间、IP地址
- **使用场景**: 订单创建、订单更新、支付回调、帖子审核、评论审核
- **状态**: ✅ 已完成

### 5.10 单元测试
- **框架**: JUnit 5 + Mockito
- **覆盖范围**: OrderService、AdminService、PostService、CommentService、UserService
- **测试类型**: 单元测试、集成测试
- **状态**: ✅ 已完成

## 7. 改进建议

#### 1. 加强安全措施

**问题分析**: 
- `UserController.loginByPhone` 方法没有验证码验证，存在安全风险
- 部分敏感接口缺少权限验证
- 用户密码和手机号等敏感信息未加密存储

**具体改进措施**:
