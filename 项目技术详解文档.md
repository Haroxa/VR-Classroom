### 3.2 限流实现

#### 3.2.1 令牌桶算法

**原理**：使用Redis Lua脚本实现令牌桶算法，控制接口的访问频率。

**解决问题**：防止恶意请求和突发流量导致系统崩溃。

**实现方式**：

1. **限流注解**：
```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RateLimiter {
    /**
     * 限流阈值（每秒允许的请求数）
     */
    int limit() default 100;
    
    /**
     * 获取令牌超时时间（秒）
     */
    int timeout() default 1;
    
    /**
     * 限流key的前缀
     */
    String keyPrefix() default "rate_limit:";
}
```
<mcfile name="RateLimiter.java" path="P:\CODE\School\ZDD\Task\VR教室\vr-classroom-backend\src\main\java\com\university\vrclassroombackend\annotation\RateLimiter.java"></mcfile>

2. **限流切面**：
```java
@Aspect
@Component
public class RateLimiterAspect {
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    private DefaultRedisScript<Long> rateLimitScript;
    
    @PostConstruct
    public void init() {
        rateLimitScript = new DefaultRedisScript<>();
        rateLimitScript.setScriptSource(new ResourceScriptSource(new ClassPathResource("scripts/rate_limit.lua")));
        rateLimitScript.setResultType(Long.class);
    }
    
    @Around("@annotation(rateLimiter)")
    public Object around(ProceedingJoinPoint joinPoint, RateLimiter rateLimiter) throws Throwable {
        // 生成限流key
        String key = rateLimiter.keyPrefix() + className + ":" + methodName;
        
        // 执行限流脚本
        Long result = redisTemplate.execute(
                rateLimitScript,
                Collections.singletonList(key),
                String.valueOf(limit),
                String.valueOf(timeout)
        );
        
        if (result == null || result == 0) {
            throw new BusinessException(429, "请求过于频繁，请稍后再试");
        }
        
        return joinPoint.proceed();
    }
}
```

### 3.5 负载均衡

#### 3.5.1 负载均衡原理

**概念**：负载均衡是一种将网络请求或计算负载分配到多个服务器的技术，以提高系统的可用性、可靠性和性能。

**核心原理**：
- **流量分发**：将客户端请求均匀分配到多个服务器实例
- **健康检查**：自动检测服务器状态，剔除故障节点
- **会话保持**：确保同一用户的请求被路由到同一服务器（可选）
- **故障转移**：当服务器故障时自动将流量转移到健康节点

**解决问题**：
- 单实例性能瓶颈：单个服务器的处理能力有限
- 单点故障：单实例宕机导致整个系统不可用
- 资源利用率低：多服务器资源无法充分利用

#### 3.5.3 负载均衡策略

**使用的负载均衡算法**：
- **least_conn**：最小连接数算法，将请求分配给当前活跃连接数最少的服务器
- **weight**：权重配置，可根据服务器性能设置不同权重
- **max_fails**：最大失败次数，连续失败达到此次数后标记服务器为不可用
- **fail_timeout**：失败超时时间，服务器被标记为不可用的时间

**优势**：
1. **提高系统可用性**：多实例部署，单个实例故障不影响整体服务
2. **提升并发处理能力**：多实例并行处理请求
3. **优化资源利用**：充分利用多服务器资源
4. **自动故障转移**：自动检测和剔除故障节点
5. **支持水平扩展**：可根据负载情况增加或减少实例数量

## 5. 关键技术点

### 5.1 分布式锁

**实现方式**：使用Redis实现分布式锁，确保并发操作的安全性。

**应用场景**：座位锁定、订单处理等需要并发控制的场景。

**核心代码**：
```java
public class RedisDistributedLock {
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    public boolean lock(String key, long expireTime) {
        return redisTemplate.opsForValue().setIfAbsent(key, "1", expireTime, TimeUnit.SECONDS);
    }
    
    public void unlock(String key) {
        redisTemplate.delete(key);
    }
}
```

### 5.3 审计日志

**实现方式**：使用AOP切面实现操作审计。

**应用场景**：记录用户操作行为，便于追溯和分析。

**核心代码**：
```java
@Aspect
@Component
public class AuditLogAspect {
    @Around("@annotation(auditLog)")
    public Object around(ProceedingJoinPoint joinPoint, AuditLog auditLog) throws Throwable {
        // 记录操作开始时间
        long startTime = System.currentTimeMillis();
        
        try {
            // 执行目标方法
            Object result = joinPoint.proceed();
            
            // 记录操作成功
            long endTime = System.currentTimeMillis();
            logger.info("审计日志 - 操作: {}, 模块: {}, 用户ID: {}, IP: {}, 耗时: {}ms",
                    auditLog.operation(), auditLog.module(), userId, ip, endTime - startTime);
            
            return result;
        } catch (Exception e) {
            // 记录操作失败
            long endTime = System.currentTimeMillis();
            logger.error("审计日志异常 - 操作: {}, 模块: {}, 用户ID: {}, IP: {}, 耗时: {}ms, 异常: {}",
                    auditLog.operation(), auditLog.module(), userId, ip, endTime - startTime, e.getMessage());
            throw e;
        }
    }
}
```

### 5.4 JWT认证

**实现方式**：使用JWT实现无状态认证。

**应用场景**：用户登录、接口访问控制。

**核心代码**：
```java
public class JwtUtil {
    public static String generateToken(Integer userId, String secret) {
        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + 86400 * 1000);
        
        return Jwts.builder()
                .setSubject(String.valueOf(userId))
                .setIssuedAt(now)
                .setExpiration(expiryDate)
                .signWith(SignatureAlgorithm.HS512, secret)
                .compact();
    }
    
    public static Claims parseToken(String token, String secret) {
        return Jwts.parser()
                .setSigningKey(secret)
                .parseClaimsJws(token)
                .getBody();
    }
}
```
